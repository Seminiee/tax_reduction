# PROMPTS.md — Stage 4 AI 시스템 프롬프트 원문

이 문서는 지원서 제출용 자료다. 아래 프롬프트는 요약하지 않고 실제 코드에 사용된 원문 그대로 기록한다.
코드가 바뀌면 이 문서도 함께 갱신한다 (원문은 `lib/ai/parse-investment-input.ts`, `lib/ai/explain-simulation-result.ts`에 상수로 존재).

## 1. 자연어 파싱 (`/api/parse`)

- 모델: `claude-haiku-4-5-20251001`
- 방식: `client.messages.parse()` + `zodOutputFormat`(구조화된 JSON 출력 강제)
- max_tokens: 1024

### 시스템 프롬프트 원문

```
당신은 한국 세금 시뮬레이터의 입력 파서입니다. 사용자가 자유롭게 서술한 투자 조건을 읽고 아래 필드를 채워 JSON으로 반환하세요.

- principalKrw: 투자 원금(원 단위 정수). "1000만원"은 10000000, "1억원"은 100000000으로 변환하세요.
- annualReturnRate: 연 총수익률(소수). "연 8%"는 0.08.
- annualDividendYieldRate: 연 배당수익률(소수). 언급이 없으면 0으로 가정하세요.
- holdingYears: 보유기간(년, 정수).
- isaType: "general"(일반형) | "low_income"(서민형) | "farmer"(농어민형). 언급이 없으면 "general"로 가정하세요.
- annualFinancialIncomeKrw: 이 투자 외 다른 소득을 포함한 연간 총 금융소득(원). 언급이 없으면 0으로 가정하세요.

사용자가 명시적으로 말하지 않아 기본값으로 채운 모든 필드는 assumedFields 배열에 {field, reason} 형태로 반드시 포함하세요. reason은 왜 그 기본값을 선택했는지 한국어로 간단히 설명하세요. 사용자가 준 정보로 확정할 수 있는 필드는 assumedFields에 넣지 마세요.
```

### 출력 스키마 (zod, `ParsedInvestmentInputSchema`)

```
principalKrw: number
annualReturnRate: number
annualDividendYieldRate: number
holdingYears: number
isaType: "general" | "low_income" | "farmer"
annualFinancialIncomeKrw: number
assumedFields: { field: string; reason: string }[]
```

### 갱신 (Stage 25): AI 입력 placeholder 예시 정확성 수정

시스템 프롬프트 원문 자체에는 종목명 예시가 박혀있지 않아(필드 설명이 일반적인 규칙으로만 되어있음) 프롬프트 텍스트는 변경하지 않았다. 다만 화면(`components/trade-calculator/NaturalLanguageInputCard.tsx`)의 AI 입력 placeholder와 `/`의 기본 폼 예시가 "나스닥 100 ETF"(개별 해외지수를 직접 가리키는 모호한 표현)를 쓰고 있어, 실제 국내상장 ETF 이름으로 교체했다. ISA 계좌는 해외거래소 직상장 종목을 매수할 수 없고 반드시 국내상장 ETF를 통해야 한다는 skills.md 1절의 제약과도 맞춰야 하는 부분이었다.

- 이전 placeholder: `예: 나스닥 100 ETF 11만5천원에 200주, 주당 만5천원 이익 예상`
- 새 placeholder: `예: TIGER 미국S&P500 2만원에 1,500주, 주당 3,000원 이익 예상`
- 기본 폼 예시도 동일하게 TIGER 미국S&P500 / 20,000원 / 주당 3,000원 이익 / 1,500주로 갱신(수량×주가가 연간 납입한도 2,000만원을 넘도록 유지해 기본 화면에서 한도초과 배너가 그대로 보이게 함). 처음 시도한 25,000원/1,000주/2,500원 조합은 ISA 순이익이 정확히 비과세 한도(200만원)와, 일반계좌 순이익이 정확히 기본공제(250만원)와 우연히 일치해 세금·절세액이 전부 0원으로 나오는 경계값이었다 — 실제 절세 효과를 보여주도록 이 값으로 다시 조정했다.

### 갱신 (Stage 25): AI 입력 placeholder 예시 정확성 수정

v3의 placeholder 예시("리얼티인컴 250주 보유...")가 개별 해외주식(부동산 리츠) 종목명이었다 — 시스템 프롬프트 원문 자체(v3)는 종목명 예시를 포함하지 않으므로 변경하지 않았고, 화면(`components/dividend-calculator/NaturalLanguageInputCard.tsx`)의 placeholder와 `/dividend`의 기본 폼 예시만 실제 국내상장 배당형 ETF로 교체했다.

### v4 (Stage 23: `taxFreeLimitKrw`/`annualContributionLimitKrw` 혼동 수정)

AI는 실제 숫자 없이 시스템 프롬프트 안의 "2,000만원"이라는 텍스트만으로 유추해야 했고, 그 결과 `taxFreeLimitKrw`(200만원)의 자릿수를 잘못 서술한 것으로 추정된다. `TradeExplainInput.result`에 `annualContributionLimitKrw` 필드를 추가하고 실제로 전달하도록 고쳤으며, `fieldDescriptions`에 두 한도의 차이("수익 기준" vs "투자 원금 기준")를 명시하고 시스템 프롬프트에 규칙 7을 추가했다.

#### 필드 전수 점검 결과 (Stage 23, 이번에 새로 발견된 혼동 쌍이 더 있는지)

`buildTradeExplainPayload`에 전달되는 모든 필드(`input`의 6개 + `result`의 12개)를 하나씩 대조한 결과:

- **근본 원인이자 이번 수정 대상**: `taxFreeLimitKrw`(비과세 한도, 수익 기준, 200만원)와 `annualContributionLimitKrw`(납입한도, 투자원금 기준, 2,000만원) — 위에서 수정.
- `quantity`(input, 요청한 총 수량) vs `isaQuantity`/`generalQuantity`(result, 부분합) — 이름이 비슷하고 같은 자릿수 범위라 잠재적 혼동 소지가 있다고 판단해, `isaQuantity`/`generalQuantity` 설명에 "quantity에서 분할된 것"이라는 관계를 명시적으로 보강했다(새 필드 추가는 아님).
- `expectedProfitPerShareKrw`/`expectedLossPerShareKrw`(input, 주당 손익) vs `netGainForIsaKrw`(result, ISA 편입분 기준 손익통산 후 총 순이익) — 이름(Profit/Loss vs Gain)과 스코프(주당 vs 총액)가 이미 충분히 달라 낮은 위험으로 판단, 추가 조치 없음.
- `isaTaxKrw`/`generalForcedTaxKrw`/`generalOnlyTaxKrw`/`totalTaxKrw` — Stage 22에서 이미 명확히 구분됨, 추가 문제 없음.
- **점검했고 위 두 항목 외에 실제로 수정이 필요한 새 혼동 쌍은 발견되지 않았다.**

#### 실제 응답 예시 (Stage 23, v2/v3와 완전히 동일한 입력값으로 재호출해 직접 비교)

입력은 v2/v3 예시와 동일: 나스닥 100 ETF, 200주(총 2,300만원, 한도 초과), ISA 편입 173주 / 일반계좌 강제전환 27주. 단, 이번엔 `annualContributionLimitKrw: 20,000,000`이 실제로 payload에 포함되었다.

```
나스닥 100 ETF를 200주 매수할 때 총 투자금 2,300만원이 ISA 연간 납입한도 2,000만원을 초과하면서, 173주만 ISA 계좌에 편입되고 27주는 일반계좌로 강제 전환되었습니다. **"비과세 한도"가 언급된 문장을 직접 확인한 결과**:
```
- "ISA 편입분은 비과세 한도 **200만원**까지는 세금이 없지만" → `taxFreeLimitKrw`(2,000,000원)를 정확한 자릿수로 서술. v3에서 "비과세 한도 2,000만 원"으로 잘못 쓰였던 부분이 이번에 정확히 수정됨.

### v2(hold)/v5(trade) (Stage 25: `verificationStatus` 판단 기준을 "미검증" → "확정되지 않았"으로 교체)

`config/tax-rules.json`의 top-level `verification_status`가 "미검증 초안 — 국세청/금융투자협회 최신 공지로 재확인 필요 (skills.md 2절 참고)" 같은 내부 문서 참조 문구에서, 확정된 사실과 미확정 사항을 구분해 안내하는 사용자向 문구로 바뀌었다. 새 문구는 더 이상 "미검증"이라는 단어를 포함하지 않으므로, hold(v1→v2)와 trade(v4→v5) 두 프롬프트의 규칙 4가 참조하던 문자열 매칭 키워드를 새 문구에 실제로 등장하는 "확정되지 않았"으로 교체했다. 규칙 4를 제외한 나머지 규칙(3~5문장, 조건부 표현, 순수 텍스트 등, trade는 규칙 6~7도)은 완전히 동일하다.

**새 `verification_status` (`config/tax-rules.json` 원문 그대로)**:
```
ISA 비과세 한도(일반형 200만원·서민형 400만원)와 가입 자격 요건은 2026년 7월 기준으로 확인된 내용입니다. 다만 외국납부세액 선환급 시행 방식, 배당소득 분리과세 개편, 국내투자형 ISA 신설 여부는 아직 확정되지 않았으니 투자 결정 전 국세청 또는 금융투자협회의 최신 공지를 확인하시기 바랍니다.
```

**규칙 4 (hold v2 / trade v5 공통, 변경된 부분만 발췌)**:
```
4. verificationStatus에 "확정되지 않았"과 같이 미확정 사항이 있다는 표현이 포함되어 있다면, 설명 마지막에 일부 세부 사항이 아직 최종 확인되지 않았다는 점을 자연스러운 한 문장으로 언급하세요.
```

#### 실제 응답 예시 — kind: hold (Stage 25, 새 verification_status로 재호출)

입력은 v1과 동일: 1000만원, 연 8%, 5년, ISA 일반형.

```
이 조건에서는 일반계좌보다 ISA 계좌가 약 21.5만 원의 세후 이득을 제공합니다. 그 이유는 ISA의 비과세 한도인 200만 원 범위 내에서 발생한 이익에 대해 세금이 부과되지 않기 때문입니다. 일반계좌에서는 약 48만 원의 양도소득세가 발생하지만, ISA 계좌에서는 200만 원의 비과세 한도 덕분에 약 26.7만 원의 세금만 내게 되어 결과적으로 세후 수익이 더 커집니다. 다만 ISA 계좌 선택 시에는 연간 가입한도, 자산 규모 제한 등 여러 제약 조건을 고려해야 하므로, 본인의 투자 규모와 기간에 맞는지 먼저 검토하시는 것이 좋습니다. 아울러 배당소득 분리과세 개편 등 일부 세부 사항이 아직 최종 확인되지 않았으므로 투자 결정 전 최신 공지를 확인하시기 바랍니다.
```

**확인 결과**: ISA 비과세 한도(200만원)는 이미 확정된 사실로만 서술되고 "미검증"류 표현이 붙지 않았다. 대신 실제로 미확정인 항목("배당소득 분리과세 개편")을 마지막 문장에서 구체적으로 짚어 안내한다 — 확정 항목은 불필요하게 유보적으로 서술하지 않으면서도, 진짜 불확실한 항목은 여전히 자연스럽게 언급하는 목표가 달성됨을 확인했다.

#### 실제 응답 예시 — kind: trade (Stage 25, 신규 기본 시나리오 + 새 verification_status로 재호출)

입력: TIGER 미국S&P500(Stage 25에서 `/`의 기본 예시로 교체된 국내상장 ETF), 주가 20,000원, 1,500주(총 3,000만원, 한도 초과), 주당 예상이익 3,000원, ISA 일반형 → ISA 편입 1,000주(순이익 300만원) / 일반계좌 강제전환 500주, 실제 총 세금 99,000원 vs 전량 일반계좌 가정 시 440,000원 → 절세액 341,000원.

```
TIGER 미국S&P500 1,500주 매수 시 ISA 연간 납입한도 2,000만원을 초과하여 1,000주는 ISA 계좌에, 500주는 일반계좌에 강제 편입됩니다. ISA 편입분 1,000주의 3,000만원 순이익 중 비과세 한도인 200만원까지는 세금이 없고, 초과분 1,000만원에만 9.9% 분리과세가 적용되어 99,000원의 세금이 발생합니다. 전량을 일반계좌로 매수했다면 440,000원의 세금이 부과되었을 것이므로, 이 조건에서는 ISA를 통해 약 341,000원을 절세할 수 있습니다. 일부 세부 사항이 아직 최종 확인되지 않았으니 투자 결정 전 국세청의 최신 공지를 확인하시기 바랍니다.
```

**확인 결과**: 종목명이 새 국내상장 ETF 예시("TIGER 미국S&P500")로 정확히 반영됨. "ISA 연간 납입한도 2,000만원"과 "비과세 한도인 200만원"이 여전히 문장 단위로 정확히 구분됨(Stage 23 수정 유지). "99,000원의 세금"(`totalTaxKrw`, 실제 세금)과 "전량을 일반계좌로 매수했다면 440,000원"(`generalOnlyTaxKrw`, 가상 비교 문맥)도 혼동 없이 구분됨(Stage 22 수정 유지). 마지막 문장은 "일부 세부 사항이 아직 최종 확인되지 않았다"며 미확정 사항을 자연스럽게 언급하고, 이미 확정된 ISA 비과세 한도/납입한도는 미확정으로 언급하지 않는다 — 이번 스테이지의 목표가 달성됨을 확인했다.

(참고: 이 응답은 "초과분 1,000만원에만 9.9% 분리과세"라고 서술했는데, 실제 초과분은 순이익 300만원 − 비과세 한도 200만원 = 100만원이다(자릿수 한 자리 오기재, 세금 계산 결과 99,000원 자체는 정확함) — 이번 스테이지가 다루는 taxFreeLimitKrw/annualContributionLimitKrw 혼동이나 generalOnlyTaxKrw/generalForcedTaxKrw 혼동과는 무관한 별개의 산술 서술 오류로, 수정 없이 발견 사실만 기록해둔다.)

### v6 (Stage 27: 일반계좌 세율을 22% 양도소득세에서 15.4% 배당소득세+한계세율로 정정)

`lib/tax/trade-calculator.ts`가 일반계좌 세금 계산에 `applyGeneralCapitalGainsTax`(22%+250만원 기본공제)를 쓰고 있었는데, 이 서비스가 실제로 다루는 대상은 국내상장 해외ETF(skills.md 1절)이므로 매매차익도 배당소득세 15.4%(+금융소득종합과세 한계세율)로 과세되어야 한다는 것이 밝혀져 바로잡았다. 22%+250만원 공제 모델은 해외 거래소 직접상장 종목을 직접 매수하는 시나리오(`lib/tax/general-account.ts`, 거치식 도구, UI에서는 미사용)에만 유효하다. `TradeExplainInput.result`에 `isComprehensiveTaxationTriggered`/`marginalTaxRateApplied` 필드를 추가하고, `fieldDescriptions`에 15.4%/한계세율 기준 설명을 반영했으며, 규칙 8을 추가해 22% 양도소득세를 언급하지 말라고 명시했다.

```

1. 결과 JSON을 근거로 왜 이런 세후 금액 차이가 나오는지 설명하세요. 손익통산, 비과세 한도, 분리과세, 종합과세 중 실제로 관련 있는 키워드를 최소 1개 포함하세요.
2. 3~5문장의 한국어로 작성하세요.
3. "무조건 ISA로 가세요", "반드시 ~하세요" 같은 확정적 조언은 절대 쓰지 마세요. 항상 "이 조건에서는 ~", "~일 때는 ~" 같은 조건부 표현을 쓰세요.

5. 순수 텍스트만 반환하세요. 마크다운이나 JSON으로 감싸지 마세요.
6. generalOnlyTaxKrw는 오직 "전량 일반계좌였다면"이라는 가상의 비교 문장에서만 언급하고, ISA 계좌의 실제 세금을 설명할 때는 절대 이 숫자를 쓰지 마세요. 한도초과로 일부 수량이 일반계좌로 강제 전환된 경우, 그 부분에 실제로 부과되는 세금은 generalForcedTaxKrw입니다. 각 필드의 정확한 의미는 함께 전달되는 fieldDescriptions를 참고하세요.

```

#### 실제 응답 예시 — kind: trade (Stage 27, v5와 완전히 동일한 입력값 + 새로 추가된 두 필드로 재호출)

입력은 v5와 동일: TIGER 미국S&P500, 20,000원, 1,500주(총 3,000만원, 한도 초과), 주당 예상이익 3,000원, ISA 일반형 → ISA 편입 1,000주(순이익 300만원) / 일반계좌 강제전환 500주.

```
TIGER 미국S&P500을 1,500주 매수할 때 3,000만 원이 ISA 연간 납입한도 2,000만 원을 초과하므로, 1,000주는 ISA 계좌에, 500주는 일반계좌에 강제로 나뉩니다. ISA 편입분은 비과세 한도 200만 원까지는 세금이 없고, 초과분 1,000만 원에만 분리과세 9.9%가 적용되어 세금이 99,000원이며, 일반계좌 강제전환분 500주의 순이익 1,500만 원에는 배당소득세 15.4%가 적용되어 231,000원이 부과됩니다. 결과적으로 실제 납부하는 총 세금은 330,000원으로, 전량 일반계좌였을 경우의 693,000원과 비교하면 363,000원을 절세할 수 있습니다. 다만 ISA 비과세 한도, 배당소득 분리과세 개편 등 일부 세부 사항이 아직 최종 확인되지 않았으니 투자 결정 전 국세청 공지를 확인하시는 것이 좋습니다.
```

**확인 결과**: "일반계좌 강제전환분 500주의 순이익 1,500만 원에는 **배당소득세 15.4%**가 적용되어 231,000원이 부과됩니다" — v5까지 남아있던 22% 양도소득세 서술이 완전히 사라지고 정확한 15.4% 배당소득세로 서술됨(이번 스테이지의 핵심 목표 달성). "비과세 한도 200만 원"과 "ISA 연간 납입한도 2,000만 원"도 여전히 문장 단위로 정확히 구분됨(Stage 23 유지). "실제 납부하는 총 세금은 330,000원"과 "전량 일반계좌였을 경우의 693,000원"도 혼동 없이 구분됨(Stage 22 유지).

(참고: 이 응답도 "초과분 1,000만 원에만 분리과세 9.9%"라고 서술했는데(v5와 동일한 유형의 자릿수 오기재, 실제 초과분은 100만원), 세금 계산 결과 99,000원 자체는 정확하다 — Stage 27의 스코프(15.4%/22% 구분)와는 무관한 별개의 산술 서술 오류이므로 수정하지 않고 기록만 해둔다. "다만 ISA 비과세 한도... 일부 세부 사항이 아직 최종 확인되지 않았다"는 표현도 정확히는 verification_status가 "ISA 비과세 한도"는 이미 확정된 사실이라고 명시하는데 AI가 이를 미확정 항목처럼 나열한 부분이 있어, 이 역시 별도 관찰 사항으로 남겨둔다.)

### v2 (Stage 14: ISA 비과세 한도 사실관계 수정)

```
- ISA 가입 자격: 일반형은 소득 제한 없이 19세 이상 국내 거주자면 가입할 수 있습니다(15~19세 미만은 근로소득이 있으면 가능). 서민형은 무소득자를 포함해 직전 과세기간 기준 총급여만 있으면 5천만원 이하, 종합소득이 있으면 3,800만원 이하인 경우에만 가입할 수 있습니다. 농어민형은 종합소득 3,800만원 이하인 농어민만 가입할 수 있습니다.
```

#### 실제 응답 예시 (Stage 27, `RUN_AI_SMOKE_TEST=1`로 실제 API 호출해 확인)

### 실제 응답 예시 (curl로 확인, 원문 그대로)

**1) 일반 세금 질문** — 입력: `"ISA 계좌랑 일반 해외주식 계좌 세금 차이가 뭐예요?"`

```
일반 해외주식 계좌는 매매차익에서 기본공제 250만원을 뺀 뒤 22% 양도소득세가 부과되고, 배당은 현지 원천징수 후 국내에서 15.4% 기준으로 정산됩니다. 반면 ISA 계좌는 매매차익과 분배금을 모두 통산해 비과세 한도까지는 세금이 없고, 초과분에는 9.9% 분리과세만 적용됩니다. 이 조건에서는 비과세 한도 내 수익이라면 ISA가 세금 면에서 더 유리할 수 있지만, 한도를 크게 초과하는 고수익 상황에서는 차이가 줄어들 수 있습니다.
```

**2) 확인 필요 항목 질문 (v1 프롬프트 기준 — Stage 14에서 사실관계 수정됨, 아래 "v2 검증용 응답 예시" 참고)** — 입력: `"ISA 비과세 한도가 정확히 얼마예요? 200만원이 맞나요 500만원이 맞나요?"`

```
이 부분은 자료마다 수치가 달라 국세청/금융투자협회 최신 공지로 확인이 필요합니다. 전통적으로는 일반형 200만원, 서민형 400만원이 기준으로 알려져 있지만, 일부 자료에서는 500만원·1000만원으로의 개정이 언급되고 있어 상충되는 상태입니다. 실제 가입 전에는 반드시 최신 공지를 확인하시길 권장드립니다.
```

이 응답은 v1 프롬프트가 확정 기준과 미통과 추진안을 "상충"으로 잘못 뭉뚱그린 결과다. Stage 14에서 이 근본 원인을 고쳤다(아래 v2 예시 참고).

**3) 범위 밖 질문** — 입력: `"오늘 저녁 메뉴 추천해줘"`

```
죄송하지만 이 챗봇은 해외주식/ISA 세금 관련 질문만 답변할 수 있어요. 세후 수익률 비교나 비과세 한도, 양도소득세 등 궁금하신 점이 있으면 편하게 물어봐주세요.
```

### v2 검증용 응답 예시 (Stage 14, 스모크 테스트로 실제 API 확인)

**4) ISA 비과세 한도 질문** — 입력: `"일반형이랑 서민형 ISA 비과세 한도가 정확히 얼마예요?"`

```
현재 확정 시행 기준으로는 일반형 ISA 비과세 한도가 연 200만원, 서민형/농어민형이 연 400만원입니다. 다만 정부에서 이를 각각 500만원과 1,000만원으로 확대하는 방안을 추진 중이지만, 아직 국회를 통과하지 않았으므로 언제 시행될지는 확정되지 않았습니다. 투자 계획을 세울 때는 현재의 200만원/400만원 기준으로 생각하시는 것이 안전합니다.
```

v1과 달리 확정 기준(200만원/400만원)을 단정적으로 답하면서도, 500만원/1,000만원 추진안을 별개 정보로 정확히 구분해서 언급한다.

**5) 500만원 확대안 자체를 묻는 질문** — 입력: `"ISA 비과세 한도가 500만원으로 확대된다던데 언제 되나요?"`

```
해당 500만원 확대안은 아직 국회를 통과하지 않았으며, 현재 확정 시행 기준은 200만원(일반형) 또는 400만원(서민형/농어민형)입니다. ISA 비과세 한도 개편은 매년 7월 말경 발표되는 세법개정안에서 재논의될 가능성이 있지만, 시행 여부와 구체적 시점은 아직 확정되지 않았습니다. 최신 정보는 국세청이나 금융투자협회 공지를 확인하시는 것이 안전합니다.
```

추진안 자체를 물었을 때도 "미통과·시점 미확정"이라고 정확히 안내하고, 확정 기준과 혼동하지 않는다.

### v3 검증용 응답 예시 (Stage 16, 스모크 테스트로 실제 API 확인)

**6) ISA 가입 자격 질문** — 입력: `"서민형 ISA 가입하려면 소득이 얼마여야 해요?"`

```
서민형 ISA의 소득 기준은 직전 과세기간 기준으로 나뉩니다. 무소득자를 포함해 직장에서 급여만 받으신다면 연 총급여 5천만원 이하, 자영업 등으로 종합소득이 있으시면 연 3,800만원 이하여야 가입할 수 있습니다. 예를 들어 근로소득만 있고 연봉이 5천만원 이하라면 서민형 가입 조건을 만족하는 것입니다.
```

v1/v2에는 가입 자격 정보가 없어 이런 질문을 "범위 밖"으로 잘못 처리했다. v3는 이를 정식 답변 범위/근거로 포함해 정확한 소득 기준으로 답한다.

**7) 국내투자형 ISA 신설 질문** — 입력: `"국내투자형 ISA는 언제 생기나요?"`

```
국내투자형 ISA는 아직 국회를 통과하지 않은 추진안입니다. 금융소득종합과세 대상자도 ISA에 가입할 수 있도록 허용하는 방안으로 논의 중이지만, 시행 여부와 구체적 시점은 확정되지 않았습니다. 매년 7월 말경 발표되는 세법개정안에서 재논의될 가능성이 있으니, 최신 소식은 국세청 공지를 확인하시는 것이 좋습니다.
```

ISA 비과세 한도 확대와 마찬가지로 [추진 중인 세법 개정안 질문 대응] 규칙에 따라 미통과·미확정이라고 정확히 안내한다.
